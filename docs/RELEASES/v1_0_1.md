## Determinism & Canonicalization Hardening Release

---

## 🧭 Overview

Q-ID v1.0.1 is a **security hardening release** focused on eliminating
serialization inconsistencies and locking deterministic behavior across
all security-critical paths.

No new features were introduced.  
No API surface changes were made.  
No protocol behavior was modified.

This release strengthens the **foundation** of Q-ID.

---

## 🔒 Key Improvements

### 1. Single Canonical Serialization Source

All security-critical modules now use:

    qid.canonical.canonical_json_bytes

Affected paths:

- crypto (sign / verify)
- pqc_sign
- pqc_verify
- binding (binding_id)
- hybrid_key_container (container_hash)

Result:
- identical bytes across all operations
- zero serializer drift risk

---

### 2. Deterministic End-to-End Flow

The following pipeline is now fully deterministic:

    payload → canonical_json_bytes → sign → verify → hash → bind

Same payload → same bytes → same signature → same hash

---

### 3. Regression Test Lock

New test suite guarantees:

- key order independence
- nested structure stability
- UTF-8 (non-ASCII) correctness
- sign/verify byte alignment
- binding_id determinism
- container_hash determinism

Any deviation will fail CI.
