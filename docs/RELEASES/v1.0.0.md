# Q-ID v1.0.0 --- Stable Contract Release

This release formalizes Q-ID as a **stable, contract-locked identity
protocol** with hardened CI, deterministic integration adapters, and
documented security guarantees.

v1.0.0 supersedes all `v0.1.x-ci-locked` tags and marks the transition
from CI-hardening phase to stable integration phase.

------------------------------------------------------------------------

## What v1.0.0 Represents

-   Stable API surface
-   Stable protocol behavior
-   Stable integration adapters (Adamantine + Guardian)
-   Deterministic fail-closed validation model
-   CI-enforced coverage gate
-   Optional real-PQC backend support (liboqs)

No breaking behavior changes relative to v0.1.2-ci-locked.

------------------------------------------------------------------------

# Release History (Pre‑1.0 Hardening Phase)

------------------------------------------------------------------------

## v0.1.0-ci-locked

Initial CI-locked contract freeze.

-   API surface contract frozen at `contracts/api_surface_v0_1.json`
-   CI: pytest + coverage gate (≥90%)
-   CI-safe stub crypto + fail-closed PQC backend selection
-   Hybrid ML-DSA + Falcon container support

Purpose: Establish deterministic contract baseline.

------------------------------------------------------------------------

## v0.1.1-ci-locked

Protocol hardening and stricter guarantees.

Changes since v0.1.0:

-   Coverage gate raised and enforced at ≥95%
-   README aligned exactly with code + test contracts
-   liboqs key generation paths fully covered and verified
-   PQC backend enforcement hardened (no silent fallback)
-   Hybrid ML-DSA + Falcon rules fully locked and tested

No API surface changes. No protocol behavior changes.

Purpose: Auditor-grade protocol hardening.

------------------------------------------------------------------------

## v0.1.2-ci-locked

Coverage policy realignment with optional PQC workflow.

Changes since v0.1.1:

-   CI coverage gate adjusted to ≥90% to reflect stub-only environments
-   PQC Optional (real liboqs backend) workflow fully green
-   No code changes to protocol, crypto rules, or guarantees
-   No API surface changes
-   No behavioral changes

Coverage rationale: Real PQC paths (ML-DSA, Falcon, Hybrid) are fully
exercised in the "PQC Optional (liboqs real backend)" workflow. Default
CI coverage reflects intentional stub-only execution environments.

Purpose: CI policy stabilization before stable release.
